SQL & NoSQL Presentation

1. What is SQL?

SQL is a language used to talk to databases. It stores data in tables with rows and columns.

2. Characteristics of SQL

  • Tells the database what you want, not how to get it.
  • Uses tables to store data.
  • Works on most databases.
  • Can add, edit, delete, or read data.
  • Can create or change tables.
  • Safe changes: can save or undo.
  • Can work on many rows at once.

3. Databases that use SQL

  • MySQL
  • PostgreSQL
  • Microsoft SQL Server
  • Oracle
  • SQLite
  • MariaDB

4. What is NoSQL?

NoSQL is a type of database that doesn’t rely on tables. It’s flexible and stores data in documents, key-values, graphs, or columns.

5. What does "No" stand for?

No means "Not only". So NoSQL = Not only SQL. Some NoSQL databases can still use tables sometimes.

6. Characteristics of NoSQL

  • Flexible data structure
  • Schema can change anytime
  • Can scale by adding servers
  • Transactions supported, but less strict than SQL
  • Handles big, fast, or changing data

7. Databases that use NoSQL

  • MongoDB
  • Redis
  • Cassandra
  • Firebase Realtime Database / Firestore
  • CouchDB
  • Neo4j
