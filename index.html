SQL & NoSQL

1. What is SQL?

SQL is a language used to talk to databases. It stores data in tables with rows and columns.

2. Characteristics of SQL

  • Uses tables: Data is stored in rows and columns, like a spreadsheet.
  • Works on most databases: Many databases like MySQL, PostgreSQL, and Oracle use SQL.
  • Can change data: You can add, update, delete, or read data easily.
  • Can create or change tables: You can make new tables or change old ones.
  • Safe changes: You can save or undo changes if something goes wrong.
  • Handles many rows at once: You can work on lots of data in one command.

3. Databases that use SQL

  • MySQL
  • PostgreSQL
  • Microsoft SQL Server
  • Oracle
  • SQLite
  • MariaDB

4. What is NoSQL?

NoSQL is a type of database that doesn’t rely on tables. It’s flexible and stores data in documents, key-values, etc.

5. What does "No" stand for?

No means "Not only". So NoSQL = Not only SQL. Some NoSQL databases can still use tables sometimes.

6. Characteristics of NoSQL

  • Flexible structure: Data can be stored in different formats, not just tables.
  • Can change anytime: You don’t need to plan the data structure before using it.
  • Scales easily: You can add more servers to handle more data.
  • Transactions: Some NoSQL databases support safe changes, but not as strict as SQL.
  • Handles big or fast data: Great for lots of data or data that changes quickly.

7. Databases that use NoSQL

  • MongoDB
  • Redis
  • Cassandra
  • Firebase Realtime Database / Firestore
  • CouchDB
  • Neo4j
