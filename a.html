1. What is SQL?

SQL (Structured Query Language) is used to manage relational databases. Data is stored in tables with rows and columns.

Example: Students Table

Name Age Grade
Leo 20 A
Amy 21 B

2. Characteristics of SQL

  • Declarative: You say what you want, not how to do it. SELECT Name FROM Students;
  • Uses Tables: Structured rows and columns. Example above.
  • Works on Most Databases: MySQL, PostgreSQL, Oracle.
  • Data Manipulation: Add, update, delete data. INSERT INTO Students VALUES('Mia',22,'A');
  • Data Definition: Create or modify tables. CREATE TABLE Students(Name VARCHAR(50), Age INT);
  • Transactional: Safe changes. BEGIN TRANSACTION; ... COMMIT;
  • Set-based Operations: Update multiple rows at once. UPDATE Students SET Grade='B' WHERE Age>20;

3. Databases that use SQL

  • MySQL
  • PostgreSQL
  • Microsoft SQL Server
  • Oracle Database
  • SQLite
  • MariaDB

4. What is NoSQL?

NoSQL databases handle unstructured or semi-structured data. They are flexible, scalable, and fast.

Examples:

Document (MongoDB):

{ "Name": "Leo", "Age": 20, "Grade": "A", "Hobby": "Coding" }

Key-Value (Redis):

"user:1001" -> "Leo"

5. What does "No" stand for?

"No" = "Not only". NoSQL means not only SQL tables; focus is on flexibility and speed.

6. Characteristics of NoSQL

  • Flexible Structure: Can store documents, key-values, graphs, columns.
  • Dynamic Schema: Add fields anytime without affecting others. { "Hobby": "Football" }
  • Scales Easily: Add servers for more data. Used by Instagram, Facebook.
  • Transaction Support: Some support multi-document transactions.
  • Handles Big/Fast Data: Ideal for live apps, analytics, chat apps.

7. Databases that use NoSQL

  • MongoDB – Document-based
  • Redis – Key-value store
  • Cassandra – Wide-column store
  • Firebase Realtime DB / Firestore – Apps
  • CouchDB – Document-based
  • Neo4j – Graph database
